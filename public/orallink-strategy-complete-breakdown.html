🎯 Skip Nesting, Use Scan ID Pop-up

The WHY, WHAT, and HOW - Complete Strategy Breakdown

🤔 WHY Skip Nesting?

Problem #1: Nesting Doesn't Actually Work

What Happens:

  1. User in iframe (orallink.health)
  2. User enters email (still in iframe)
  3. OralLink redirects to Kells
  4. Redirect BREAKS THE IFRAME
  5. User on Kells (full page, no iframe)
  6. After scan: Dead end (same problem)
Result: Nesting works for 30 seconds, then fails. You get all the complexity, none of the benefits.

What Happens:

  1. User clicks "Get AI Scan"
  2. Pop-up shows Scan ID
  3. Direct redirect to orallink.health
  4. User completes scan on Kells
  5. User returns, enters Scan ID
  6. Chatbot resumes with context
Result: Same return rate as nesting (40-50%), but simpler code and honest UX (user knows they're leaving).

🚫 The Fatal Flaw of Nesting

When OralLink redirects to Kells (window.location.href = "getkells.com"), the redirect ALWAYS navigates the top-level window, not just the iframe. This is by design in web browsers - you cannot prevent it without breaking the user flow.

Your options:

  • Block redirect with sandbox: User can't reach Kells → Can't complete scan → 100% failure
  • Allow redirect: Iframe breaks, user leaves site → Same abandonment problem

Problem #2: Mobile Camera Restrictions

📱 iOS Safari (50% of your traffic)

Apple restricts camera access in iframes for security reasons. Even if nesting worked, users couldn't take photos on iPhone/iPad. This is a hard browser limitation you cannot overcome.

Problem #3: Complexity vs Benefit

Aspect With Nesting Without Nesting
Dev Time 3-5 days (iframe setup, detection, messaging) 1 day (pop-up + database)
Return Rate 40-50% (same as simple redirect) 40-50% (same!)
Code Complexity High (iframe communication, detection) Low (just pop-up + input)
Mobile Works? No (camera blocked in iframe) Yes (direct redirect = full page)
Debugging Painful (cross-origin issues) Simple (just track IDs)
User Expectation Confused (seems integrated, then breaks) Clear (pop-up sets expectation)

✓ Bottom Line

Nesting delivers 5x complexity for 0x benefit. You get the same return rate (40-50%) whether you nest or not, because the iframe breaks anyway when OralLink redirects to Kells. Why build complex infrastructure that doesn't improve outcomes?
