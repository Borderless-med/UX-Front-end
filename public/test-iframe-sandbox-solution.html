🧪 Iframe Sandbox Test

Testing if sandbox="allow-same-origin allow-scripts allow-forms allow-popups" prevents redirect escape

Test 1: Regular Iframe (Control Group)

Code: <iframe src="test-page.html"></iframe>
Behavior: Redirect will escape iframe and navigate parent window

⚠️ Expected Result

When you click "Redirect to Google" inside iframe, the ENTIRE page (parent window) navigates to Google. Iframe "escapes".

Test 2: Sandboxed Iframe (Suggested Solution)

Code: <iframe sandbox="allow-same-origin allow-scripts allow-forms allow-popups"></iframe>
Behavior: Redirect is BLOCKED (no allow-top-navigation)

❌ Expected Result

When you click "Redirect to Google" inside iframe, nothing happens or you see browser error: "Navigation blocked by sandbox". User is STUCK - cannot proceed to external site (Kells).

Test 3: Sandbox WITH allow-top-navigation

Code: <iframe sandbox="allow-same-origin allow-scripts allow-forms allow-popups allow-top-navigation"></iframe>
Behavior: Same as Test 1 (redirect escapes)

⚠️ Expected Result

Adding allow-top-navigation permission = back to square one. Redirect escapes iframe just like Test 1. Sandbox provides no benefit.

🚫 The Fundamental Problem

Your user flow REQUIRES the redirect to Kells:

  1. User enters email on orallink.health (in iframe)
  2. OralLink redirects to getkells.com (NECESSARY for scan)
  3. User takes photos and submits scan on Kells
  4. User needs to return to OraChope somehow

Sandbox breaks step 2: If you block redirects, users cannot reach Kells to complete scan. The entire flow dies.

📱 About the "Header Section" Comment

The suggestion mentions "mobile should work if we have a header section". This likely refers to keeping your OraChope header visible above the iframe. While this provides visual continuity, it does NOT solve:

  • Camera permission issues: iOS Safari restricts camera access in iframes regardless of headers
  • The redirect problem: Headers don't prevent iframe content from redirecting parent window
  • The dead end: After Kells redirect, header disappears anyway (user left your site)
