⚙️ How It Works

1 User clicks "Get FREE AI Scan" in chatbot
2 Generate unique Scan ID: OL-[year]-[random]
3 Store in database: scan_id, user_session, timestamp, status='initiated'
4 Show pop-up with Scan ID (big, copyable)
5 User copies/screenshots ID, clicks "Continue"
6 Redirect: window.location = "orallink.health?ref=" + scan_id
7 User completes scan on Kells → DEAD END
8 User manually returns to OraChope (hours/days later)
9 Chatbot asks: "Did you complete scan? Enter Scan ID:"
10 Lookup scan_id in database → fetch scan details
11 Chatbot: "I see you need [treatments]. Let me find clinics..."

📊 Pros & Cons Analysis

✓ PROS

  • Zero cost (no Kells dev fees)
  • You control entire UX
  • No dependency on OralLink
  • Can track scan initiations
  • Simple to implement (1 day)
  • Can add email follow-up later
  • Clear user expectation (they know they're leaving)

✗ CONS

  • 40-50% abandonment rate (users forget/lose ID)
  • High friction (manual ID entry)
  • Looks unprofessional/clunky
  • User frustration ("why can't it be automatic?")
  • No way to verify scan actually completed
  • Lost users = lost bookings
  • Reflects badly on OraChope brand

⚡ Expected Performance

If 1,000 users click "Get AI Scan":

  • 1,000 see Scan ID pop-up
  • 850 proceed to OralLink (15% cancel at pop-up)
  • 600 complete scan on Kells (30% drop-off during scan)
  • 300 return and enter Scan ID (50% abandonment - forget/lose ID)
  • 60 book clinic after returning (20% conversion)

Bottom line: 300 lost users who completed scan but never returned = 60 lost bookings.
