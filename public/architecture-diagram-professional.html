System Architecture Diagram – Multi-Channel Dental Clinic Platform

Architecture Components Explained

- Dual Role) 1) Tool Gateway: Enables AI apps (Claude, ChatGPT) to access dental platform tools. 2) LLM Abstraction: Allows swapping LLM models (Gemini, GPT-4, Claude) without changing backend code - just update configuration Enables AI applications to access our dental platform using Model Context Protocol. Shares the same business logic flows as the web app, ensuring consistency.
React Frontend Traditional web interface for users to search clinics, view details, and book appointments via browser.
Backend API Server Core business logic and orchestration. Shared by both web and MCP channels to ensure consistent behavior.
LLM Models & AI Services Currently using Google Gemini 2.0 Flash and gemini-embedding-001. Can be swapped for GPT-4, Claude, or other models via MCP Server configuration without code changes.
Database Stores clinic information, patient reviews, and vector embeddings for semantic search capabilities.
Client Applications Two access methods: Traditional web browsers and AI chat applications (Claude, ChatGPT) via MCP protocol.
