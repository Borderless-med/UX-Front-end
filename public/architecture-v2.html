System Architecture Diagram – Multi-Channel Dental Clinic Platform

Architecture Components Explained

ALL chatbot traffic routes here - both your website's embedded chatbot AND external AI apps (Claude, ChatGPT)
MCP Server (Dual Role - Key Innovation) 1) Tool Gateway: Connects AI apps (Claude, ChatGPT) to our dental platform backend. Provides tools like answer_dental_question and find_clinic. 2) LLM Abstraction Layer: Sits between backend and AI services. Allows swapping LLM models (Gemini ↔ GPT-4 ↔ Claude) by just updating configuration - no code changes needed!
Web interface with forms AND embedded chatbot widget. The chatbot widget connects to MCP Server for AI-powered conversations
React Frontend Traditional web interface for users to search clinics, view details, and book appointments via browser.
Backend API Server Core business logic orchestration in main.py and flows/ directory. Shared by both web chatbot and MCP channels. Uses MCP Server's LLM Gateway for AI model access.
LLM Models & AI Services MCP Server acts as intermediary - flows request AI through MCP's LLM Gateway (gemini_service.py), which can swap models (Gemini ↔ GPT-4 ↔ Claude) via configuration.
Database Stores clinic information, patient reviews, and vector embeddings for semantic search capabilities. Uses pgvector extension for similarity search.
Client Applications Two access methods: Traditional web browsers and AI chat applications (Claude, ChatGPT) via MCP protocol. Same backend serves both!
